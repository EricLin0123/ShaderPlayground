# Creative Studio Technical Content Grand Plan

## Goal
Publish one technical post per week (Twitter + Threads) that demonstrates studio capability and creative taste.

## Editorial Principles
- Tone: clear, confident, technical, curious.
- Depth: practical know-how with one key insight per post (no deep tutorial overload).
- Structure per post: hook -> concept -> mini breakdown -> visual/demo -> takeaway.
- Proof of capability: every post should reference a real experiment, snippet, animation, or visual output.

## Content Pillars (Big Topics -> Small Topics)

### 1) CSS Motion & Layout Systems
- Advanced layout composition (grid overlays, asymmetry, layered UI)
- Scroll-linked and timeline-driven animation
- SVG + CSS hybrid motion
- Performance-aware motion design

### 2) JavaScript Interaction Engineering
- Interaction architecture (state, timing, events)
- Input-driven animation (pointer, scroll, velocity)
- Motion systems (spring, damping, interpolation)
- Production-safe animation patterns

### 3) WebGL & Shader Craft
- Shader fundamentals for designers
- Noise, distortion, gradients, procedural texture
- Post-processing and compositing
- Interactive shader scenes for product/web storytelling

### 4) Creative 3D for Web Experiences
- Camera language and scene composition
- Material/light cheats for web performance
- Hybrid DOM + 3D composition
- Narrative-driven 3D interaction

## Lightweight Weekly Writing Template (Use every week)
- Hook: one surprising visual/technical claim.
- Context: where this technique fits in real projects.
- Core idea: one principle + one implementation detail.
- Evidence: short clip/GIF + 5-15 lines of code or pseudo-code.
- Close: one practical takeaway and teaser for next week.

## Distribution Workflow (per post)
- Twitter: concise thread (3-6 tweets), code-first language.
- Threads: more narrative version with process framing.
- Visual asset: 1 short loop + 1 still breakdown.
- CTA: invite collaboration or link to studio capability page.

## Consistency Guardrails
- Keep jargon controlled; define advanced terms in one line.
- No post without visual proof.
- No post without practical takeaway.
- Maintain recurring identity: same voice, same visual signature, same technical rigor.
